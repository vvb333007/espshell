⇮SKETCH VARIABLES, REGISTERING

ESPShell has API to register and manipulate sketch variables, although variable types are limited to simple C types (unsigned/signed int, char, short, float or generic void * pointer). Arrays of abovementioned simple types are supported as well

In order to make an arbitrary sketch variable to be visible in the ESPShell one has to register it.

Registration is done with convar_add() macro. There are also convar_addp() and convar_adda() etc macros for registering variables, pointers and arrays.

Typical use in Arduino sketch: 

  static int some_variable;
  unsigned char *pointer;
  char array[10];

  void setup() {

    convar_add(some_variable); < - Register simple type variable
    convar_addp(pointer);      < - Register pointer to a simple type
    convar_adda(array);        < - Register array of simple types
  }
Other convar_add macros are: convar_addap(Name) - register an array of pointers, convar_addpp(Name) - register a pointer to a pointer

⇮DISPLAYING VARIABLES LIST AND VALUES

There are 3 different variables defined in the code snippet above: int, pointer and an array. All three are registered with appropriate convar_addX() macros and after this is done these variables become accessible from the shell by using "var NAME" and "show memory ADDRESS" commands

Executed without arguments, "var" command displays list of registered variables: 

    esp32#>var
    % Sketch variables:
    % Variable X name | sizeof(X) |     typeof(X)    |     Value
    %-----------------+-----------+------------------+----------------
    %           table |         4 |            int * |       0x3ffbdb88
    %              a4 |         4 |          float * |       0x3ffbdbb4
    %              a3 |         4 |            float |        10.000000
    %              a2 |         4 |     unsigned int |                0
    %              a1 |         1 |    unsigned char |               99
    %              a0 |         4 |              int |              -99
    %     tbl_min_len |         2 |   unsigned short |               16
    %       bypass_qm |         4 |              int |                0
    %       pcnt_unit |         4 |              int |                0
    %    pcnt_channel |         4 |              int |                0
    %ls_show_dir_size |         4 |              int |                1
    esp32#>

Note that for pointers and arrays their address is displayed instead of memory content pointed to. To dereference a pointer or to display an array content one can use "var VAR_NAME": 

    esp32#>var a4
    % float * a4 = 0x3ffbdbb4;  // Pointer to 4 bytes memory region
    % a4[1] = {
    %    10.000000, // a4[0]
    % };

Another example: 

    esp32#>var table
    % int * table = 0x3ffbdb88;  // Array of 10 elements, (4 bytes per element)
    % int table[10] = {
    %    1, // table[0]
    %    2, // table[1]
    %    3, // table[2]
    %    4, // table[3]
    %    5, // table[4]
    %    6, // table[5]
    %    7, // table[6]
    %    8, // table[7]
    %    9, // table[8]
    %    0, // table[9]
    % };

To access an array element simply add its index like in C language. 

    esp32#>var table[5]
    % int table[5] = 6;
    esp32#>

Note that this syntax can also be used to access memory regions defined by pointers, not arrays. Using an index for a pointer variable simply means adding index value to the pointer before dereferencing. There will be warning message displayed in such case because ESPShell have no idea on memory ranges

In the same way other variables can be displayed: 

    esp32#>var a1
    % unsigned char a1 = 99;
    esp32#>

⇮SETTING NEW VALUE TO THE VARIABLE

To set a variable to new value one can use "var VARIABLE_NAME NEW_VALUE" command. Lets set value of variable a1 to 88: 

    esp32#>var a1 88
    esp32#>var a1
    % unsigned char a1 = 88;  // use "var 88" see this number in hex, oct, bin etc
    esp32#>

There are some basic type checking which forbids setting unsigned variables to signed values or assign floating point values to integer variables. Also, changing a value of an array has no effect. Address of array is defined by linker and can not be changed in runtime

⇮NUMBER BASE CONVERSION

Another use of "var" command is to convert & cast numbers: if argument to the "var" command is a number (i.e. decimal, hexadecimal, floating point, octal or binary) then a table is displayed showing the number in different bases plus, an unsafe C-style memory cast is performed; main use of this "C-cast" is to see hex representation of a negative or floating point numbers

Example: 

                          
    esp32#>var 666
    % "666" is a number, which can be written as:
    % C-style cast of a memory content:
    % unsigned : 666
    %   signed : 666
    % float    : 0.000000
    % Same number in different bases:
    % Hex      : 0x29a
    % Octal    : 01232
    % Binary   : 0b1010011010
    esp32#>

Or 

    esp32#>var 666.666
    % "666.666" is a number, which can be written as:
    % C-style cast of a memory content:
    % unsigned : 1143384736
    %   signed : 1143384736
    % float    : 666.666016
    % Same number in different bases:
    % Hex      : 0x4426aaa0
    % Octal    : 010411525240
    % Binary   : 0b1000100001001101010101010100000
    esp32#>

Or 

    esp32#>var 0b111
    % "0b111" is a number, which can be written as:
    % C-style cast of a memory content:
    % unsigned : 7
    %   signed : 7
    % float    : 0.000000
    % Same number in different bases:
    % Hex      : 0x7
    % Octal    : 07
    % Binary   : 0b111
    esp32#>
